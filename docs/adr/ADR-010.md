# ADR-010: AI Agent Onboarding und dokumentationsgetriebene Projektuebergabe

## Problem
Kuenftige KI-Agenten muessen sich aktuell Wissen aus mehreren Stellen zusammensuchen:
README, Master-Handbook, Logbook, Changelog und ADRs.
Ohne klaren Einstiegspunkt steigt das Risiko, dass eine KI zu frueh im Code arbeitet, den dokumentierten Projektstand falsch einschaetzt oder Prioritaeten und Architektur aus dem Code statt aus der verbindlichen Dokumentation ableitet.

## Entscheidung
Jarvis bekommt zwei zusaetzliche Governance-Dokumente:

- `docs/AI_START.md` als verpflichtenden Einstiegspunkt fuer jede KI.
- `docs/PROJECT_STATE.md` als kurze, eindeutig pflegbare Zusammenfassung des aktuellen Projektstands.

`docs/AI_START.md` legt die verpflichtende Lesereihenfolge, die Entscheidungsgrenzen der KI und die Pflicht fest, vor jeder Aenderung zuerst den dokumentierten Projektstand zu beantworten.
`docs/PROJECT_STATE.md` fasst den laufenden Stand aus Handbook, Logbook, Changelog und ADRs zusammen, ohne selbst neue Roadmap-Punkte zu erfinden.

Die Hierarchie fuer Konfliktfaelle wird explizit dokumentiert:
`Handbook > ADR > Code > README`

## Begruendung
- Das Master-Handbook definiert Jarvis bereits als dokumentationsgetriebenes Projekt und als Single Source of Truth.
- KI-Agenten wechseln haeufiger als menschliche Projektbeteiligte; ein klarer Einstieg reduziert Reibung und Fehlentscheidungen bei der Uebergabe.
- Eine knappe Zustandsdatei beschleunigt die Orientierung, ohne das Handbook zu ersetzen.
- Die Regeln schuetzen davor, dass eine KI Architektur oder Roadmap stillschweigend aus vorhandenen Codefragmenten ableitet und damit am Product Owner vorbei entscheidet.

## Alternativen erwogen
- **Jede KI orientiert sich nur durch Codeanalyse:** verworfen. Der Code zeigt Implementierungsstand, aber nicht automatisch Prioritaeten, verworfene Alternativen, offene Handbuch-Punkte, Produktrollen oder dokumentierte Entscheidungen. Das widerspricht dem dokumentationsgetriebenen Ansatz von Jarvis.
- **Nur README erweitern:** verworfen. Das README ist ein Einstieg fuer Menschen, aber kein ausreichend strikter Governance-Startpunkt fuer wiederholbare KI-Uebergaben.

## Konsequenzen
- Neue KI-Agenten duerfen nicht direkt mit Implementierung beginnen, sondern muessen zuerst den dokumentierten Stand lesen.
- `docs/PROJECT_STATE.md`, `docs/logbook.md` und `docs/CHANGELOG.md` werden Teil der verpflichtenden Pflege nach jeder Aenderung.
- Die Code-Architektur aendert sich durch diese ADR nicht; geaendert wird die Projekt-Governance fuer dokumentationsgetriebene Uebergaben.

## Status
**Superseded** (Governance-Umbau, 2026-07-03). Der verpflichtende Einstiegspunkt ist ab jetzt `CONTRIBUTING.md` (Jarvis Developer Charter); dort leben Lesereihenfolge/Session-Runbook, die Entscheidungsgrenzen (Delegations-Matrix) und die Dokument-Hierarchie. `docs/AI_START.md` bleibt nur als Weiterleitung auf `CONTRIBUTING.md` bestehen. `docs/PROJECT_STATE.md` bleibt gültig (jetzt mit maschinenlesbarem Kopf). Der dokumentationsgetriebene Grundgedanke dieser ADR gilt unverändert weiter. Details: `CONTRIBUTING.md`, `docs/logbook.md` (2026-07-03). Keine neue ADR (reine Governance-/Prozessarbeit, PO-Entscheidung).
