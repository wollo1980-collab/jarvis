# ADR-042: News-Briefing über RSS-Feeds (get_news)

## Problem/Kontext

Nutzungslauf-Befund (2026-07-09): Auf „gibt es Nachrichten auf der Welt?"
wählte der Planner korrekt `search_web` — aber Suchtreffer für solche Fragen
sind **Portale** (tagesschau.de, spiegel.de) mit generischen Beschreibungen,
keine Meldungen. Gesprochen las Jarvis minutenlang Website-Empfehlungen samt
URLs vor. Für „was ist heute los?" ist eine Websuche das falsche Werkzeug;
der PO wünscht das Butler-Verhalten: „3–4 Top-News erzählen."

## Entscheidung

1. **Neuer read-only Connector `core/news_reader.py`:** liest Schlagzeilen
   aus RSS-2.0-/Atom-Feeds — **stdlib-only** (urllib + xml.etree), kein
   API-Key, kein Scraping, kein neues Paket. Fetcher injizierbar (Tests ohne
   Netz). Ein kaputter Feed wird geloggt und übersprungen; Meldungen werden
   reihum über die Feeds verteilt (keine Quelle dominiert).
2. **Neuer Command `get_news`** (Stufe 0, keine Bestätigung): Top-N
   Schlagzeilen (Default 4, Deckel 8) mit Ein-Satz-Beschreibung und Quelle,
   im Persona-Ton („Die Lage, Sir — …"). In der Standalone-Whitelist (rein
   lesend, wie `check_mail`).
3. **Feeds konfigurierbar:** `news_feeds` in der Config (Default:
   tagesschau); beliebig erweiterbar, keine Codeänderung nötig.
4. **Planner-Abgrenzung:** `get_news` = Schlagzeilen-Briefing („was gibt's
   Neues?"); `search_web` = gezielte Recherche zu einem konkreten Thema.

## Begründung

RSS ist der seit Jahrzehnten stabile, maschinenlesbare Weg an Schlagzeilen —
deterministisch, kostenlos, ohne Abhängigkeit. Die Lösung folgt exakt dem
Muster der bestehenden Connectoren (Mail ADR-031, Web ADR-032): read-only,
lokal verarbeitet, injizierbar getestet.

## Alternativen erwogen

- **search_web-Ergebnisseiten nachladen und Schlagzeilen extrahieren:**
  verworfen — Scraping ist fragil, rechtlich grauer, teurer.
- **News-API (z. B. NewsAPI.org):** verworfen — API-Key, Kontingente,
  Abhängigkeit; RSS liefert dasselbe frei Haus.
- **LLM aus eigenem Wissen antworten lassen:** verworfen — veraltet
  garantiert.

## Konsequenzen

- „Was gibt's Neues?" liefert echte Meldungen — per Telegram und gesprochen
  (die Sprechfassung des PTT-Kanals hält es zusätzlich kurz).
- Feed-Inhalte sind Fremdinhalte: sie werden nur angezeigt/vorgelesen, nie
  als Anweisungen interpretiert (reine Daten im Result-Text).

## Falsifizierbarkeit

Zu revidieren, falls (a) relevante Quellen RSS abschalten, (b) der PO
themenspezifische Briefings braucht (dann Feeds-pro-Thema oder Recherche-
Verknüpfung) oder (c) die Rundum-Verteilung unpassende Gewichtung ergibt.

## Bewusst NICHT in diesem Schritt

Themenfilter/Personalisierung · Zusammenfassung per LLM (Kosten; Roh-
Schlagzeilen genügen) · Caching · geplantes Morgen-Briefing (Kandidat für
A2-Ausbau: Trigger × get_news) · weitere Default-Feeds (PO erweitert selbst).

## Status

Accepted (PO-Freigabe 2026-07-09, Doppel-Scheibe mit der Sprechfassung des
PTT-Kanals).
