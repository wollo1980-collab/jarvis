# ADR-054: Impuls-Kreislauf - Jarvis' proaktiver Herzschlag

## Problem/Kontext

PO-Ziel (11.07.2026, [Angestellten-Vision]): Jarvis soll zum proaktiven
Endsystem werden, wie es die Branche 2026 anpeilt ("zwei Drittel der
Nutzer wollen eine KI, die Beduerfnisse vorwegnimmt, ohne gefragt zu
werden"). Bis heute ist Jarvis fast rein REAKTIV: er antwortet gut,
beginnt aber nichts. Es fehlt der eine Baustein, der aus vorhandenen
echten Quellen (Wetter, Gewohnheits-Statistik ADR-053, Delegations-Logs)
von selbst Relevantes VORLEGT.

## Entscheidung

Ein **Impuls-Kreislauf** auf dem bestehenden Scheduler-Thread (ADR-039):

1. **Engine + Pruefer (`core/impulses.py`):** Die `ImpulseEngine` fuehrt in
   festem, langsamem Takt (900 s) injizierte Pruefer aus. Ein Pruefer ist
   eine reine Funktion `() -> list[dict]`, die eine ECHTE Quelle liest und
   Impuls-Kandidaten liefert (kind/key/title/detail). Erster Pruefer:
   Unwetter (`make_weather_checker`) aus dem gecachten Wetterabruf.
2. **Speicher + Entprellung + Nein-Liste (`memory/impulses.py`):** Der
   `ImpulseStore` persistiert offene Impulse und dedupliziert per `key`;
   tagesbezogene Keys tragen das Datum (`weather-storm-2026-07-11`), damit
   morgen ein frischer Anlauf moeglich ist. Ein weggeklickter Impuls landet
   in `dismissed` (Nein-Liste) - derselbe key kommt nicht wieder.
3. **Still als Karte:** Ein Impuls wird NICHT gesprochen (PO-Entscheidung
   11.07.: "lieber als Karte, nicht selbststaendig reden"), sondern
   erscheint als Dashboard-Karte (`dashboard_data.open_impulses`, ganz oben,
   ✕ zum Wegklicken).
4. **Wegklick UI-intern (`commands/impulses.py` + `/impulse/dismiss`):**
   eine Modul-Funktion, KEIN Registry-Command - damit der Wegklick nie im
   Planner-Prompt landet und die KI ihn nie aus einer Sprach-/Telegram-
   Eingabe waehlen kann (fail-closed, wie /entry/delete).

## Begruendung

Alle Leitplanken existierten schon: Vorschlag-statt-Aktion, still statt
gesprochen (PO), das Bestaetigungs-Gate fuer alles Handelnde. Der
Impuls-Kreislauf fuegt kein neues Risiko hinzu - er ist die Engine, die
vorhandene Karten von "wird beim Laden berechnet" auf "Jarvis hat an dich
gedacht" hebt. Ein Impuls hat keinen Arm, nur eine Stimme.

## Leitplanken (eisern)

- **Vorschlag statt Aktion:** ein Impuls informiert/fragt, fuehrt nie aus.
- **Nur echte Quellen:** nie eine erfundene Stimmung.
- **Gedeckelt:** hoechstens `_MAX_OPEN_IMPULSES` offen; nachts (22-6 Uhr)
  wird nichts gelegt.
- **Weggeklickt = verstanden:** Nein-Liste verhindert Wiederkehr.
- **Fail-safe:** ein kaputter Pruefer stoppt die anderen nie; der Kreislauf
  stoert nie den Erinnerungs-Scheduler oder die Runtime.

## Alternativen

- **Gesprochene proaktive Meldung zu fester Uhrzeit:** vom PO ausdruecklich
  ABGELEHNT (11.07.) - stille Karte gewuenscht.
- **Eigener Thread:** verworfen - der Scheduler-Thread existiert und pollt
  bereits; ein zweiter Thread waere unnoetige Nebenlaeufigkeit.
- **Impuls-Wegklick als Registry-Command:** verworfen - erschiene im
  Planner-Prompt und waere aus Sprache waehlbar; UI-interne Funktion ist
  fail-closed.

## Konsequenzen

- Neue Datei `memory_data/impulses.json` (offen + Nein-Liste, bounded).
- Neues Config-Feld `impulses_enabled` (Default AN - der PO hat das
  proaktive Endsystem ausdruecklich beauftragt; abschaltbar).
- Der Scheduler-Thread laeuft jetzt auch ohne Push-Kanal (fuer Impulse).
- Folge-Pruefer (Gewohnheits-Frage ADR-053, Agenten-Ergebnis) docken ohne
  Architekturaenderung an - nur neue Pruefer-Funktionen.

## Falsifizierbarkeit

Gilt als falsch, wenn Impulse als aufdringlich empfunden werden (PO-Nerv-
Effekt, Kill-Kriterium der Vision) - dann Takt/Deckel/Ruhefenster
nachziehen oder Pruefer abschalten, nie die Stille-/Vorschlags-Grundsaetze
aufweichen.

## Bewusst NICHT

- Kein gesprochener Impuls, keine autonome Handlung.
- Kein generischer Befehls-Endpunkt (nur der eine harte /impulse/dismiss).
- Keine Impuls-Quelle ohne echte Datengrundlage.

## Status

Accepted (PO-Auftrag 11.07.2026: "Setz das alles um. Jarvis soll zum
Endsystem werden wie es auch Google etc. plant." - erste Kampagnen-Scheibe).
